# Math_Collatz

Math_Collatz is not intended as a general mathematics.

It is a local language generated by the Collatz rule.

## Accelerated odd transition

For positive odd n_i:

    3 n_i + 1 = 2^(a_i) n_(i+1)

where:

    a_i = v2(3 n_i + 1)

and n_(i+1) is the next odd term.

## Debt word

A Collatz odd trajectory generates a debt word:

    W(n_0) = (a_0, a_1, a_2, ...)

The partial debt is:

    A_k = a_0 + a_1 + ... + a_(k-1)

After k odd blocks, the main multiplicative factor is approximately:

    3^k / 2^(A_k)

Escape requires:

    A_k / k <= log2(3)

Compression dominates when:

    A_k / k > log2(3)

## 2-adic shadow

Define:

    s_i = v2(n_i + 1)

This measures how close n_i is to -1 in the 2-adic sense.

If:

    s_i >= 2

then:

    a_i = 1

and:

    s_(i+1) = s_i - 1

Therefore:

    escape consumes shadow.

## Regeneration

If a later block increases s, shadow is regenerated.

A dangerous trajectory must repeatedly regenerate enough shadow to create more a_i = 1 steps.

This motivates the core target:

    no infinite cheap regeneration chain
