# Audit Boundary

Version: v4.3

The audit is a repository consistency layer.

It is not a theory layer.

It is not a proof layer.

It is not a theorem layer.

## Correct interpretation

audit_passed means:

The checked repository artifacts are consistent with the native-language boundary.

## Incorrect interpretation

audit_passed does not mean:

Collatz solved.

Proof found.

Global closure established.

Global invariant established.

Obstruction impossible.

## Core rule

No solution before native language.
