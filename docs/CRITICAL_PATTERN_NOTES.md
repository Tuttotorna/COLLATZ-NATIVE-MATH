# Critical Pattern Notes

This document records the working interpretation of the v1.0 critical case.

The key event is not merely a long trajectory.

The key event is a low-debt post-chain recovery window that remains under the escape threshold for many blocks, then barely crosses above it.

In native terms:

    escape pressure is not eliminated immediately
    compensation accumulates slowly
    crossing occurs only after a long window
    the final surplus is small

The useful next question is:

    can the same pattern be generalized,
    bounded,
    or made arbitrarily long?

If it can be made arbitrarily long without recovery, the current CRC idea fails.

If it cannot, the obstruction may become a real lemma candidate.
