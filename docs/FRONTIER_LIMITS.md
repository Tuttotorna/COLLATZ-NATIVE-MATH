# Frontier Scan Limits

The v1.1 frontier scan is deliberately bounded.

It cannot prove convergence.

It cannot rule out harder cases outside the scanned set.

It can only say:

    within this candidate frontier,
    this is the hardest observed recovered post-chain case.

The value is operational:

    it prevents the project from overfitting one famous case
    it tests whether the critical pattern repeats nearby
    it produces the next target automatically if a harder case appears

A proof would require a bound on the frontier itself.

v1.1 does not claim that bound.
