# Learning Collatz Grammar

Version: v3.1

This document defines the research process after the route correction.

## Research posture

The repository must learn Collatz grammar before attempting to solve Collatz.

The grammar is learned by observing what the transformation produces.

## Questions to ask

Ask:

What structures appear?

Which structures persist?

Which structures disappear?

Which structures regenerate?

Which structures compensate?

Which structures fail to preserve themselves?

Which structures close?

Which structures would count as obstruction?

## Questions to defer

Defer:

Can we prove the conjecture?

Can we solve Collatz?

Can we show all numbers reach 1?

Can we announce global closure?

These questions are not deleted.

They are deferred to the translation layer.

## Grammar-first sequence

The corrected sequence is:

1. native language
2. generated structures
3. structural grammar
4. obstruction and closure meanings
5. evidence instrumentation
6. reversible translation
7. possible proof attempt

## Why this matters

If standard proof comes too early, it forces Collatz into an external frame.

If native grammar comes first, the later proof attempt may preserve the structure instead of replacing it.

## Current status

The repository has already built useful instrumentation.

v3.1 tells that instrumentation what it is for.

It is for learning Collatz as a native dynamical language.
