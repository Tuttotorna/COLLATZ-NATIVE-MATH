# Native Conservation Map

Version: v3.8

This document defines the Native Conservation Map.

After recurrence, mutation, and stability, the next native question is conservation.

Not:

Can we solve Collatz?

But:

What remains conserved across recurrence, mutation, and stability?

## Core rule

No solution before native language.

## Conservation

Conservation means that a native behavior remains visible across transformation layers.

It is not proof.

It is not theorem.

It is native persistence across grammatical change.

## Distinctions

recurrence = what returns

mutation = what changes

stability = what resists change

conservation = what remains through change

## Why this matters

If danger appears but release is conserved, that is a native fact.

If danger appears but obstruction is not conserved, that is a native fact.

If instability itself is conserved, that is a native fact.

The project must read these facts before translating them into proof language.

## Boundary

Bounded conservation signals do not imply global theorem or global closure.
