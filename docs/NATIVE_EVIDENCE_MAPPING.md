# Native Evidence Mapping

Version: v1.9

This document defines how existing computational evidence is mapped into native objects.

The purpose is not to add another bigger scan.

The purpose is to prevent methodological drift.

The previous artifacts are valid evidence layers, but they must now be read through the native language established in v1.6, v1.7, and v1.8.

## Core rule

Evidence must be mapped into native objects before standard mathematics re-enters.

The native objects are:

- debt
- shadow
- compensation
- regeneration
- closure
- obstruction

## Why this layer matters

Without this layer, the project risks jumping from finite scans directly back into classical proof language.

That would repeat the earlier deviation.

The correct order is:

native method
native obstruction
native closure
native evidence mapping
native-to-standard translation

## Evidence is not proof

A finite certificate can show:

- selected bad windows recovered
- selected adversarial candidates did not produce obstruction
- selected frontier cases remained stable
- selected hardness metrics identify stress points

But finite evidence cannot by itself show global closure.

## Correct interpretation

The evidence layer supports the native research path.

It does not replace the native closure mechanism.

## Incorrect interpretation

Do not read the evidence layer as:

- proof of Collatz
- global closure
- exhaustive obstruction absence
- standard theorem
- final solution

## v1.9 role

v1.9 translates the repository's existing artifacts into native-language status labels.

This prepares the project for v2.0, where standard translation may begin under strict boundary conditions.
