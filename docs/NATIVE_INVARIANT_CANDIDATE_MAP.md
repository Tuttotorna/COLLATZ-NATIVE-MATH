# Native Invariant Candidate Map

Version: v3.9

This document defines the Native Invariant Candidate Map.

After conservation, the next native question is invariant candidacy.

Not:

Can we prove Collatz?

But:

Which conserved behaviors are strong enough to become native invariant candidates?

## Core rule

No solution before native language.

## Invariant candidate

An invariant candidate is not an invariant theorem.

It is a conserved bounded behavior that may deserve stronger native testing.

## Distinctions

recurrence = what returns

mutation = what changes

stability = what resists change

conservation = what remains through change

invariant candidate = what may remain under repeated native tests

## Boundary

A bounded invariant candidate is not a global invariant.

A global invariant is not claimed in v3.9.
