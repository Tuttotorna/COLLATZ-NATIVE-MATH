# Native Language Inversion

Version: v3.1

This document corrects the project direction.

The project must not begin with the posture:

solve Collatz

That posture is already external. It asks Collatz to satisfy a human terminal demand.

The corrected posture is:

learn what Collatz is before asking how to solve it

The central inversion is:

Do not solve first.
Describe first.
Listen first.
Learn the native language first.

## Core correction

The classical question is:

Do all positive integers eventually reach 1?

This question is legitimate, but it is not native.

It already frames Collatz as a conjecture, a target, a terminal problem, and an object to be closed by an external observer.

The native question is earlier:

What structural language is generated by the Collatz transformation itself?

Before Collatz becomes a conjecture, it is a dynamical object.

Before it is asked to terminate, it generates structure.

Before it is asked to justify itself as true or false, it produces a grammar.

## Native starting point

The native starting point is not:

prove termination

The native starting point is:

observe the structures generated by alternating odd expansion and binary discharge

This produces a language of:

expansion
discharge
debt
shadow
regeneration
compensation
closure
obstruction

These are not metaphors. They are native structural names for behavior produced by the transformation.

## Route correction

Previous layers from v1.6 through v3.0 remain useful, but v3.1 reclassifies them.

They are not a march toward solving Collatz.

They are instruments for learning the native language of Collatz.

Proof obligations are not the center.

The center is native description.

## Mother rule

No solution before native language.

No proof before native description.

No terminal pressure before structural listening.

## Repository identity

This project is not first a proof attempt.

It is a native structural reading of the Collatz dynamics.

It asks:

What is Collatz before it becomes a conjecture?
