# Native Regeneration

Version: v1.7

Regeneration is the reappearance of dangerous debt structure after apparent recovery.

It is central because local compensation is not enough.

## Why regeneration matters

A bad window can recover locally.
But if the same trajectory produces new debt structures indefinitely, local recovery alone does not establish closure.

The native method must therefore distinguish:

- local compensation
- repeated compensation
- stable closure
- regeneration without closure

## Regeneration event

A regeneration event occurs when a trajectory produces a new debt structure after a prior debt structure has been compensated.

In native terms:

    debt -> compensation -> renewed debt

This is not automatically bad.
It is bad only if the renewed debt becomes self-preserving and avoids forced discharge.

## Benign regeneration

Benign regeneration occurs when renewed debt keeps being compensated.

This is what current finite scans suggest in selected candidate sets.

## Dangerous regeneration

Dangerous regeneration occurs when renewed debt becomes harder to erase, while compensation margins shrink or disappear.

Dangerous regeneration is the path toward obstruction.

## Native regeneration chain

A regeneration chain is a sequence:

    D1 -> C1 -> D2 -> C2 -> D3 -> C3 -> ...

where D means debt and C means compensation.

A harmless chain has repeated closure pressure.

An obstruction chain would require:

    D1 -> D2 -> D3 -> ...

without adequate compensating C.

## Research target

The next native goal is to determine whether regeneration necessarily creates opportunities for discharge.

If regeneration itself forces future compensation, then obstruction becomes structurally unstable.
