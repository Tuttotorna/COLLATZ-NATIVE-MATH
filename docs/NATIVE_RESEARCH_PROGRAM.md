# Native Research Program

Version: v1.6

This document defines the next research direction.

## Status

The repository currently contains several computational evidence layers:

- frontier stability certificate
- compensation law candidate scan
- adversarial compensation scan
- hardness metric report

These are useful.

But they are not the native method itself.

They are finite evidence layers.

## Problem with the current direction

The project began to drift back toward classical mathematical framing too early.

That drift is not fatal.

But it must be corrected by separating the native line from the evidence line.

## Corrected direction

The next phase should not add another bigger scan immediately.

The next phase should define the native obstruction model.

## Research sequence

v1.6 establishes the method boundary.

v1.7 should define the Native Obstruction Model.

v1.8 should define Native Closure Conditions.

v1.9 should map computational evidence to native objects.

v2.0 should begin the standard mathematical translation.

## Central obstruction question

Can there exist an infinite debt structure that never generates compensating discharge?

This question must be studied natively first.

Only after that should it be translated into standard proof language.

## What would count as progress

Progress is not merely finding larger hard numbers.

Progress is:

- clearer native primitives
- sharper obstruction definitions
- distinction between local recovery and global closure
- regeneration analysis
- finite-to-general transition rules
- proof boundary discipline

## What would count as failure

The native method fails if it cannot produce a structural statement stronger than finite scanning.

The computational evidence is strong as evidence.

But the native method must eventually explain why compensation is forced, not merely observe that it occurred in selected finite cases.

## Current conclusion

The project should proceed through native structure first.

The standard mathematical layer should be delayed until the native closure mechanism is explicit.
