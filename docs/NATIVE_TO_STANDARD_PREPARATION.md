# Native-to-Standard Preparation

Version: v1.9

This document prepares the next step without performing it yet.

## Current state

The repository now has:

- native method boundary
- native obstruction model
- native closure conditions
- native evidence mapping

This is enough to prepare standard translation.

But standard translation should still be controlled.

## What v2.0 should do

v2.0 should define a boundary for translating native objects into standard mathematical objects.

It should not immediately claim a theorem.

It should define translation pairs.

Examples:

native debt -> standard deficit condition

native compensation -> standard recovery inequality

native shadow -> standard persistence condition

native obstruction -> standard non-compensating admissible structure

native closure -> standard impossibility of persistent non-compensating structure

## What v2.0 should not do

v2.0 should not claim proof.

v2.0 should not erase the native language.

v2.0 should not reduce the project back to ordinary Collatz wording too early.

## Correct next question

Can each native object be translated into a standard object without losing the structure that made the native method useful?

That is the v2.0 question.
