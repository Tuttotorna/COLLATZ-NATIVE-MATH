# Native-to-Standard Translation Boundary

Version: v2.0

This document defines the controlled boundary where standard mathematics may re-enter the project.

The native method remains the source layer.

The standard layer is a translation layer.

The purpose of v2.0 is not to claim a proof. The purpose is to prevent translation from destroying the native structure.

## Translation rule

A standard mathematical statement is allowed only when it preserves a native object.

Every standard phrase must map back to one of these native objects:

- expansion
- discharge
- debt
- bad window
- shadow
- regeneration
- compensation
- surplus
- obstruction
- closure

If a standard statement cannot map back to a native object, it is not part of the v2.0 translation boundary.

## Reversibility rule

Translation must be reversible.

Correct:

native debt -> standard deficit condition -> native debt

Incorrect:

native debt -> generic inequality -> loss of native meaning

The translation is valid only if the native meaning survives the standard expression.

## What v2.0 does

v2.0 creates a bridge.

It does not replace the native method.

It does not reduce the project to the classical Collatz framing.

It does not claim that finite evidence proves the conjecture.

## What v2.0 prevents

v2.0 prevents premature collapse into:

- stopping time only
- parity iteration only
- classical convergence only
- isolated inequalities without native interpretation
- computational evidence treated as proof

## Correct order

The correct order remains:

native primitives
native obstruction
native closure
native evidence mapping
controlled standard translation
formal proof attempt

## Core claim

Standard mathematics may now re-enter, but only as a controlled translation of native structure.
