# No Solving Posture

Version: v3.1

This document defines the forbidden posture.

## Forbidden posture

The repository must not begin from:

I want to solve Collatz.
I want to force termination.
I want to prove the conjecture first.
I want to close the object before understanding its language.

This posture is external and terminal.

It treats Collatz as a problem subordinated to human closure-pressure.

## Correct posture

The repository begins from:

What does Collatz generate?
What does Collatz preserve?
What does Collatz lose?
What does Collatz regenerate?
What does Collatz close?
What does Collatz prevent?
What does Collatz call impossible according to its own dynamics?

## Why solving is deferred

Solving is not forbidden forever.

Solving is deferred because premature solving collapses the native language into classical human framing.

A proof can come later as translation.

But the source layer must be native description.

## New rule

Do not solve first.

Do not prove first.

Do not terminate first.

Do not force Collatz to answer a human question before learning the question generated by Collatz itself.

## Status

This is a route correction.

It does not invalidate previous evidence.

It repositions previous evidence as language-learning instrumentation.
