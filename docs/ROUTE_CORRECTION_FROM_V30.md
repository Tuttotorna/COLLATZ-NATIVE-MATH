# Route Correction from v3.0

Version: v3.1

This document explains how v3.1 corrects the direction after v3.0.

## What happened

The project correctly built a native vocabulary and then began translating it into standard lemma drafts.

That translation was useful, but it risked returning too quickly to the classical posture:

solve
prove
terminate
close the conjecture

## Correction

v3.1 does not delete v3.0.

It reclassifies v3.0 as a deferred translation layer.

Formal lemma drafts are not the center.

They are translation instruments that may become useful only after the native language is sufficiently understood.

## New priority

The new priority is:

Collatz as native language

not:

Collatz as problem to solve

## Reclassification

Previous evidence layers become:

native instrumentation

Previous lemma drafts become:

translation candidates

Previous proof obligations become:

translation debts

Previous obstruction scans become:

grammar probes

## Meaning

The repository is now protected from the wrong interpretation.

It is not an attempt to prove Collatz first.

It is an attempt to learn what Collatz is before proof-pressure is allowed to dominate.
