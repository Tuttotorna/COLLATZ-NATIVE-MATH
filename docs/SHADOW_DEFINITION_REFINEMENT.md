# Shadow Definition Refinement

Version: v2.3

This document refines the native definition of shadow.

Shadow is not simply the existence of a prior bad window.
Shadow is not simply the memory of a computation.
Shadow is not merely long trajectory behavior.

Shadow is the persistence of obstruction-relevant debt influence after local recovery.

## Refined native definition

A debt interval I casts shadow into a later region L when all of the following hold:

1. I created positive cumulative debt.
2. I was locally compensated.
3. Later behavior still carries obstruction-relevant structure traceable to I.
4. The later structure is not fully independent of I's debt pattern.
5. The influence can participate in renewed bad-window formation or delayed closure.

In short:

shadow = persistence of obstruction-relevant prior debt influence

## What shadow is not

Shadow is not:

- high stopping time;
- large odd-block count;
- any later bad window;
- any later fluctuation;
- proof of obstruction;
- proof of non-closure.

Shadow must preserve relevance to prior debt.

## Shadow carrier

A shadow carrier is the later trajectory structure through which prior debt remains active.

A carrier may appear as:

- repeated deficit patterns;
- shrinking compensation margins;
- regenerated bad windows;
- delayed erasure of prior imbalance;
- dependence of later stress on earlier debt.

## Shadow erasure

Shadow is erased when later structure no longer preserves obstruction-relevant dependence on prior debt.

Shadow erasure requires more than local recovery.

Local recovery says:

the bad window was compensated.

Shadow erasure says:

the prior debt no longer acts as a carrier for obstruction potential.
