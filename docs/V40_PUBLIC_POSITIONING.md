# v4.0 Public Positioning

Version: v4.0

This project should not be presented as another attempt to solve Collatz first.

It should be presented as:

A native structural reading of the Collatz dynamics.

Or:

A project that does not begin by trying to solve Collatz, but by asking what Collatz is when described through structures generated by its own transformation rule.

## Short position

Collatz is not forced to justify itself as a conjecture before it is understood as a native dynamical object.

## One-line rule

No solution before native language.

## What this protects

This protects the project from premature proof pressure.

It also prevents the native language from collapsing back into classical terminal framing too early.
